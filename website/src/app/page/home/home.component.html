The data and configuration of your shop are placed on the Blockchain. It will be resolved via the shop's URL. That's all that is needed to display your shop to your customer.

Your customer can be assured that the digital content they buy will be there forever, and can be re-downloaded all the time.

Digital Collectables?! NFTs supercharged.

Every sold digital good is represented via an digital collectiable token that governs the access to the digital file via the Lit Protocol.
Purchased items can be re-sold and form a secondary market, just like physical goods!

Even the ownership of your shop is engraved into a special Ownership-NFT. If someone else should take over ownership of your shop: Just send or sell him your ownership token.

We Layer 2! Cheap and simple.

To keep the costs low the software utilizes Layer 2 technology to execute all the Smart Contract logic inside a rollup. This reduces energy consumption substantially and makes it environment-friendly. Usability and sustainability are the number one design priority of this software.

How does it work?

This software is client-side-only. No server infrastructure is required for this shops to work. In theory this page can be self hosted or run on a home PC locally. You only need to interact with several Blockchains in order to fetch and process the shops information.

After you visit a shop via its ID the Smart Contract on Arbitrum is queried and the shop configuration is fetched from Arweave and loaded. The shop visitor can then look up items and put them into a cart. During checkout, a payment transaction is built and sent for verification, together with the money to a Smart Contract in Arbitrum. If the payment succeeds an NFT is minted that represents ownership of this digital item.

The buyer can then download the encrypted data from Arweave and decrypt it with the help of the Lit Protocol. The usage of the Arweave Blockchain makes sure encryption and data retrieval are possible for every buyer, even if the item is resold later on.

Ownership of a Shop is also expressed and controlled via a special Ownership-NFT that is minted and given to you when you create your own shop instance. If you possess this NFT you have full control over the shop and can withdraw the earned money or put items in it to sell.

FAQ

How much does it cost to operate a shop?

When you modify some shop configurations the data has to be persisted and put also into the blockchain network. This costs money. How much is a bit dependent on the load on the network. However upcoming changes to Ethereum can reduce those costs significantly. In the table below you see some experience values during low/moderate network load.

Operation Price (@ 1 ETH = 1200 $)
Create a shop 0.30 $
Edit shop configuration 0.04 $
Create item to sell 0.03 $ + 0.002 $/MB
Buy Process 0.05 $

Will the code be open source?

Yes! As soon as the codebase has matured and we are on the Beta, the code will be open sourced on Github under the GPL v3.

Will it support other rollup techs like zkSync?

Definitely. The choice fell to Arbitrum because it was pretty established and had a good support from other tools that are required for the shop to work (Lit Protocol and Bundlr Network). As soon as those support zkSync too this rollup can be supported as well.

How can this dApp help to earn crypto when you need crypto to create a shop?

Good point. Bootstrapping yourself on the blockchain can be hard if you have 0 ETH at the beginning. Thats not very welcoming to new users. One idea would be to have some sort of sponsored shops where others pay for setting up your shop for a small interest fee that needs to be payed back by the shop owner. But lets discuss more of those ideas on our group at Orbis.

Why is only Metamask wallet supported?

The goal is to be as a proof of concept to be as decentralized as possible. WalletConnect requires a connection to a backend like Infura (technically this is also true for a vanilla Metamask), with an API key. This means the dApp operator needs to keep this working. Thats what should be avoided with this project.

Do I loose access to my bought items when the shop goes offline?

The shopw software is hosted via IPFS. So anyone could host it. As long as one person on the planet hosts the website its up. Its code is also fully opened sourced. In the future there might be even offline variants of the shop that run locally. Its very unlikly that this frontend will ever go away.
The shop contracts are immutable and stay forever in the blockchain.

If items can be resold, after someone saw the content, isn't this a stupid system?

Well, in a way its very similar to the "old" way of selling digital content. A customer could also make a private copy of a CD-ROM or a casset tape and re-sell this item. In the real world you some wear and tear and people might wanted to buy a new item. Here this is mitigated by trying to enforce a royality fee. For interactive contents like a video game or computer programm you can always do some checks during startup e.g. if the user currently owns a token instance that grants access to the software.
