Why is a wallet required?

From a user perspective, it would be nice to display the shop without any connected wallet. However important information about the shops initial data is inside a Smart Contract and must be read to properly display the shops content.
A wallet is used to query this information from the chain initially to bootstrap the discovery process.

Is there no work-around?

It might be possible to run a specialied/centralized proxy server as fallback that will query the necessairy shop information from a smart contract and just forwards it to the website. The wallet requirement could then be only a fallback solution in case this server is down.

Which Wallet?

One of the mostly used wallets is the Metamask Browser extension. Later in time the shop will be hopefully compatible with Wallet Connect.
